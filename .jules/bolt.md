## 2024-05-18 - Hoisting Invariant String Operations\n**Learning:** The application heavily relies on scanning all journal records during search and score calculation. When user query string processing, such as regex splits and set allocations, is done per journal inside a map filter loop, it results in thousands of redundant operations causing noticeable bottlenecks on large datasets.\n**Action:** Always pre-calculate and hoist invariant operations (like query normalizations, string splitting, Set creation) OUTSIDE of mapping/filtering loops to ensure O(1) processing for these operations instead of O(N).
